이미지 분석: 입력 이미지의 캐릭터 구조(몸통, 꼬리, 지느러미 등)를 분석해.
프레임 생성: 분석된 구조를 바탕으로 꼬리 흔들기, 부드러운 호흡 같은 동작을 포함하는 8개의 고유한 비트맵 프레임을 생성해.
출력: 각 프레임이 연결된 스프라이트 시트와 실제 애니메이션 미리보기를 보여줘.

==

클로드, 방금 만들어준 애니메이션은 픽셀이 그냥 고무줄처럼 늘어나거나 찌그러져(Stretch/Skew) 보여서 도트 특유의 느낌이 다 망가졌어. 내가 원하는 건 **'완벽한 픽셀 아트(Pixel-perfect) 프레임 바이 프레임 애니메이션'**이야. 아래 규칙을 엄격하게 지켜서 스프라이트 시트를 다시 만들어 줘.

[필수 규칙]

변형 금지 (No Interpolation / No Transform): 절대 이미지를 회전시키거나, 크기를 늘리거나 줄이는 방식을 쓰지 마. 픽셀 크기가 달라지거나 뭉개지는 현상(Anti-aliasing, Mixel)이 1픽셀도 발생해선 안 돼.

도트 단위 재작업 (Frame-by-frame pixel shifting): 꼬리를 흔들거나 숨을 쉬는 동작을 만들 때, 해당 부위의 픽셀 덩어리를 1~2픽셀씩 정확히 상하좌우로만 이동시켜 줘.

외곽선 및 빈 공간 수정: 픽셀 덩어리를 이동시킨 후 끊어지는 외곽선(Outline)이나 빈 공간은 기존 그림체와 색상 팔레트를 그대로 사용해서 네가 직접 도트를 찍어(Redraw) 자연스럽게 이어줘.

유지 사항: 원본 이미지의 해상도(Grid)와 도트 비율을 100% 똑같이 유지할 것.

이 규칙을 바탕으로 물개가 자연스럽게 숨을 쉬며 꼬리를 까닥거리는 대기(Idle) 동작을 투명 배경의 스프라이트 시트로 정확하게 도트를 찍어내 줘.

==

 CSS의 animation과 steps() 속성을 활용해서 픽셀 아트 특유의 느낌을 살려줘. 프레임 속도(fps)를 적절히 조절해서 '대기(Idle)' 상태일 때는 자연스럽게 꼬리를 흔들거나 숨을 쉬는 것처럼 보이고, '이동(Move)' 상태일 때는 파닥거리는 모션이 자연스럽게 연결되도록 CSS 클래스를 작성해 줘.
내 작업 상태에 따라 다른 애니메이션을 보여주도록 JS와 CSS를 연동해 줘.

평소(idle): 제자리에서 부드럽게 위아래로 둥둥 떠다님

작업 중(working): 좌우로 살짝씩 빠르고 경쾌하게 흔들림 (파닥파닥)

완료(done): 제자리에서 한 바퀴 빙그르르 돎 (rotate 360deg)
각 상태를 CSS 클래스로 정의하고, JS에서 상태 텍스트가 렌더링 될 때 애니메이션 클래스도 부드럽게 전환(Transition)되도록 코드를 작성해 줘."
